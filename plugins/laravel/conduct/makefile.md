# Makefile rules

Every project must have a `Makefile` in the root with standard targets. This ensures consistency across all services and makes it easy for both developers and CI/CD pipelines.

## Required targets

### Build and run

```makefile
.PHONY: build run run-dev

# Build backend and frontend artifacts
build:
	composer install --no-interaction --prefer-dist --optimize-autoloader
	pnpm install --frozen-lockfile
	pnpm build
	php artisan config:cache
	php artisan route:cache
	php artisan view:cache

# Run production workers/processes
run:
	php artisan queue:work --tries=1

# Run local development processes
run-dev:
	composer run dev
```

### Testing

```makefile
.PHONY: test test-feature test-coverage

# Run default tests
test:
	php artisan test

# Run feature tests
test-feature:
	php artisan test --testsuite=Feature

# Run tests with coverage report
test-coverage:
	php artisan test --coverage --min=0
```

### Code quality

```makefile
.PHONY: lint fmt

# Run lint checks
lint:
	pnpm exec eslint resources/js --ext .js,.vue

# Format code
fmt:
	vendor/bin/pint --dirty
	pnpm exec prettier --write resources/js
```

### Generated artifacts

```makefile
.PHONY: optimize-clear storage-link

# Clear framework caches generated by optimize/cache targets
optimize-clear:
	php artisan optimize:clear

# Ensure public storage symlink exists
storage-link:
	php artisan storage:link
```

### Database migrations

```makefile
.PHONY: migrate-up migrate-down migrate-new migrate-status

# Run pending migrations
migrate-up:
	php artisan migrate --force

# Rollback last migration batch
migrate-down:
	php artisan migrate:rollback --step=1

# Create new migration file
migrate-new:
	php artisan make:migration $(NAME)

# Show migration status
migrate-status:
	php artisan migrate:status
```

### Frontend routing

```makefile
.PHONY: ziggy

# Generate Ziggy routes for frontend usage
ziggy:
	php artisan ziggy:generate
```

### Docker

```makefile
.PHONY: docker-build docker-up docker-down

# Build Docker image
docker-build:
	docker build -t $(APP_NAME):$(VERSION) .

# Start local environment
docker-up:
	docker compose -f docker-compose.local.yml up -d

# Stop local environment
docker-down:
	docker compose -f docker-compose.local.yml down
```

### OpenAPI

```makefile
.PHONY: openapi

# Export OpenAPI docs
openapi:
	php artisan scramble:export
```

### Documentation

```makefile
.PHONY: docs

# Show application/system documentation summary
docs:
	php artisan about
```

### Dependencies

```makefile
.PHONY: deps-install deps-update deps-audit

# Install PHP and Node dependencies
deps-install:
	composer install --no-interaction --prefer-dist
	pnpm install --frozen-lockfile

# Update PHP and Node dependencies
deps-update:
	composer update
	pnpm update

# Check dependencies for known vulnerabilities
deps-audit:
	composer audit
	pnpm audit
```

## Variables section

Every Makefile should start with configurable variables:

```makefile
APP_NAME ?= myapp
VERSION  ?= $(shell git describe --tags --always --dirty)
ENV      ?= local
PHP      ?= php
PNPM     ?= pnpm
```

## Pre-commit target

Run all quality checks before committing. This target chains the essential checks in the correct order:

```makefile
.PHONY: pre-commit

# Run all pre-commit checks: format, lint, test, build
pre-commit: fmt lint test build
	@echo "Pre-commit checks passed"
```

If API contracts or routes changed, also run `make openapi` and `make ziggy` before committing.

## CI/CD target

For CI pipelines, provide a single target that runs all checks:

```makefile
.PHONY: ci

# Full CI pipeline: lint + test + build
ci: lint test build
	@echo "CI passed"
```

## DO / DO NOT

**DO:**
- keep targets alphabetically grouped by section
- use `.PHONY` for all targets that don't produce files
- use `?=` for variables so they can be overridden: `make test ENV=staging`
- add a `help` target as the default target
- keep build targets aligned with Laravel/Inertia conventions (`composer`, `php artisan`, `pnpm`)
- keep deployment/runtime concerns explicit (`build` artifacts, cache warming, worker command)

**DO NOT:**
- put complex logic in Makefile — if it's more than 3 lines, make a script in `scripts/`
- hardcode environment-specific values — always use variables
- run broad test/build targets when a narrower target is enough for local validation
- mix local dev server concerns with Nginx/PHP-FPM deployment concerns in the same target
